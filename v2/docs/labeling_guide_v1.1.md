## 1. 라벨 정의 — 2개 라벨만 수동

### 1.1 `relevance` (정수 0/1/2)

| 값 | 정의 | 예시 판단 기준 |
|---|---|---|
| **2** | 정책 직접 기사 | 정책명·약칭·1차 키워드가 본문에 명시되고, 정책 자체의 내용·영향을 다룸 |
| **1** | 정책 배경/산업 관련 기사 | 정책 직접 언급은 약하지만, 정책 발표 시기에 같은 산업·종목 또는 정책 유발 효과를 다룸 |
| **0** | 무관/잡음 기사 | 정책과 무관한 사건·일정·정치 공방·다른 산업 기사 |

**판단 우선순위:**
1. 정책 약칭/1차 키워드 본문 매칭 → `2`
2. 정책 산업·대표종목 매칭 + 정책 효과 어휘 동반 → `1`
3. 그 외 → `0`

### 1.2 `stance_label` (범주 support/contradict/neutral)

`stance_label`은 **정책의 검증된 시장 반응 방향 `D_i`를 기준**으로 판단. 라벨링 직전 반드시 정책 카드의 `D_i_initial` 확인.

| 값 | 정의 |
|---|---|
| **support** | 기사가 `D_i` 방향의 시장 반응을 설명하거나 그 방향의 정책 전달경로 근거를 제시 |
| **contradict** | 기사가 `D_i` 반대 방향을 설명하거나 정책의 의도와 다른 시장 반응 근거를 제시 |
| **neutral** | 정보성·판단불가·양면적·정책 효과에 대한 입장 없음 |

**판단 우선순위:**
1. `D_i = +` 정책: 호재성 근거(비용 절감, 수혜, 시장 안정 등) → support, 악재성 근거 → contradict
2. `D_i = −` 정책: 악재성 근거(매출 위축, 규제 강화, 충격 등) → support, 호재성 근거 → contradict
3. 사실 전달·일정 안내·정치 공방만 있고 시장 방향 근거 없음 → neutral
4. 같은 기사 안에 양방향 근거가 비등하게 있음 → neutral

## 2. 라벨링 순서 (5단계)

```
1. 정책 카드 확인
   policy_cards_v1.1.csv에서 해당 pn 행 열기
   - alias, type_subtype, D_i_initial, active_channels, rationale

2. 자동 생성 가설 참조
   policy_hypotheses_v1.1.csv에서 해당 pn 행
   - hypothesis_support / hypothesis_contradict / hypothesis_neutral
   가설문은 "이 기사가 어느 라벨에 해당하는가"의 기준 문장

3. 기사 제목 + 본문(200자) 읽기

4. relevance 먼저 판단
   - relevance = 0 이면: stance_label = neutral 자동 (또는 비워두고 자동 처리)
   - relevance ≥ 1 이면 다음 단계

5. stance_label 판단
   - D_i 방향과 비교하여 support / contradict / neutral 중 1개 선택
   - 확신 없으면 neutral
```

## 3. 어려운 케이스 처리 매뉴얼

라벨러 판단이 흔들리는 다섯 가지 케이스. **개인 판단이 아니라 아래 룰 따름.**

### 3.1 공매도 금지 (B_market_stabilizer)
- 단어는 "규제·금지"지만 정책 카드의 `type_subtype = B_market_stabilizer`, `D_i = +`
- 기사가 **시장 안정·유동성·투자심리 개선** 어휘 → support
- 기사가 **거래 위축·시장 왜곡** 어휘 → contradict
- 기사가 단순 공매도 금지 발표 보도 → neutral

### 3.2 의대증원·정원 발표 임박 (B_restriction_negative, D_i=−)
- 기사가 정원 확대 발표 시점·규모만 알리고 **주가/시장 방향 근거가 본문에 없음** → neutral
- 기사가 의정 갈등·전공의 사직·바이오 투자심리 위축 등 **하락 근거**를 제시 → support (D_i=− 방향 지지)
- 기사가 의료 시스템 확충 등 **상승 근거**를 제시 → contradict

### 3.3 외국 정책 (E_foreign_benefit / E_foreign_shock)
- **한국 산업 관점**에서 판단 (정책 발표국 관점 아님)
- 미국 IRA (E_foreign_benefit, D_i=+): 한국 배터리 수혜 → support, 한국 부담 → contradict
- 일본 수출규제 (E_foreign_shock, D_i=−): 한국 반도체 충격 → support, 우회/대안 확보 → contradict

### 3.4 정보성 기사 (사실 전달만)
- 정책 발표 시점·규모·문구만 있고 **시장 방향 근거가 본문에 없음** → neutral
- "K칩스법이 오늘 발의된다" (정보성) → neutral
- "K칩스법으로 삼성 4조 세부담 감소 전망" (방향 근거) → support

### 3.5 종목명만 등장 (정책 키워드 없음)
- 본문에 대표 종목명만 있고 정책 직접 어휘 없음 → relevance = 1 (배경), stance는 본문 톤으로 판단
- 종목명도 정책 키워드도 없음 → relevance = 0

### 3.6 동정·일정·회의 참석·인사 기사 (v1.1 신설)

> **정책명·키워드가 본문에 있어도, 동정·일정·회의 참석·인사 기사처럼 정책 내용이나 시장 영향이 다뤄지지 않으면 relevance=1, stance=neutral로 둔다.**

- 예: "○○ 청장, 정책 가이드라인 검토 WHO 회의 참석" → relevance=1, stance=neutral (정책명 명시되나 동정성)
- 예: "장관 브리핑 사진" → relevance=2 가능하나 stance=neutral (방향 근거 없음)
- 핵심: 정책 키워드 매칭 ≠ 정책 자체의 내용·영향을 다룸. 후자가 없으면 relevance=1, stance=neutral 우선
- 근거: 파일럿 #34 (pn=34 시밀러 가이드 / 윤여표 청장 동정) 불일치 분석

### 3.7 배경 안정화 조치 (v1.1 신설)

> **정책 직접 효과가 아니라 시장환경·배경 안정화 조치만 언급된 경우, 구체적 주가/업종/정책 전달경로 근거가 없으면 stance=neutral을 우선한다. 다만 자금지원·유동성 공급·시장안정 효과가 구체적으로 제시되면 support 가능.**

- 예: "미 금융시장 안정화 조치 배경" (공매도금지 직접 언급 없음) → stance=neutral
- 예: "코로나 피해 6000억 긴급 금융지원" (100조 패키지 직접 언급 X, 유동성 공급 구체) → stance=support
- 핵심: "배경" vs "구체적 정책 전달경로 근거" 구분. 모호하면 neutral 우선.
- 근거: 파일럿 #16 (pn=13 공매도 금지 1차 / 유가 급등 + 안정화 조치) 불일치 분석

## 4. 정책 카드 참조법

각 기사 행에는 `pn`(정책 연번)이 명시되어 있음. 라벨링 시 다음 두 파일을 동시에 열어둘 것:

- **[policy_cards_v1.1.csv](policy_cards_v1.1.csv)** — 50건 카드 (`type_subtype`, `D_i_initial`, `active_channels`, `rationale`)
- **[policy_hypotheses_v1.1.csv](policy_hypotheses_v1.1.csv)** — 50건 × 3가설 자동 생성

라벨링 시 흐름:
1. 기사 행의 `pn` 확인
2. 카드 행 펼쳐서 `D_i_initial`·`active_channels`·`rationale` 1줄 읽기
3. 가설 3개 중 어느 것이 기사와 가장 가까운지 판단 → support / contradict / neutral

## 8. FAQ — 자주 묻는 질문

**Q1. relevance = 0이면 stance는 어떻게?**
A. neutral로 자동 처리. 또는 비워둬도 됨 (집계 시 neutral로 변환).

**Q2. 기사가 정책 호재인데 어조가 비판적이면?**
A. **어조가 아니라 시장 영향 방향**으로 판단. 호재 근거 제시면 support (D_i=+).

**Q3. 본문이 너무 짧아 판단 불가?**
A. 200자cap이라 흔함. 제목 + 본문 합쳐도 방향 근거 없으면 neutral.

**Q4. 정책 카드의 `D_i_initial`이 틀려 보이면?**
A. **개인 판단으로 바꾸지 말 것.** 노트에만 기록. 카드 동결본은 §1.4.3 원칙에 따라 변경 불가. 추후 서강대 CAR 확정 시 `D_i_final_car`와 대조.

**Q5. 라벨링 시간은?**
A. 1건당 평균 60~90초 목표. 100건 = 약 2시간/라벨러. 파일럿 30건 = 약 40분.
